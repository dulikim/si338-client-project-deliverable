1. Mobile-First Design

10 pts

The entire site is built as a mobile-only design. Base styles target mobile viewports and there are zero width-based media queries (min-width, max-width). The layout renders identically regardless of screen size — always the mobile view.

Key Implementation Details

  • Single-column vertical layout — All content stacks in a single column using flex-direction: column, optimized for thumb-scrolling on mobile.
  • Sticky horizontal-scrolling navigation — The nav bar uses position: sticky; top: 0; with overflow-x: auto so tabs can be swiped on narrow screens. Scrollbars are hidden via scrollbar-width: none and ::-webkit-scrollbar { display: none }.
  • Touch-friendly tap targets — Buttons, links, and cards have generous padding (minimum 44×44px effective area) for comfortable mobile tapping.
  • Responsive typography with clamp() — Every font size scales fluidly with the viewport via clamp(min, preferred, max), maintaining readability across all device widths without media queries.
  • Table reflow to cards — Tables convert to stacked card layouts where each row is a card and each cell is labeled via ::before pseudo-elements. The <thead> is visually hidden (still accessible to screen readers).
  • Horizontal scroll-snap gallery — Gallery images use CSS Grid with scroll-snap-type: x mandatory for a swipeable carousel experience.
  • No max-width anywhere — Images use width: 100% with object-fit: cover instead of max-width.

Innovative Techniques

  1. CSS Custom Properties for theming — 20+ design tokens in :root for colors, spacing, radii, shadows, and timing. Dark mode overrides only the variables.
  2. Responsive clamp() typography — Font sizes scale fluidly: --fs-base: clamp(0.9rem, 3vw, 1rem).
  3. 3D card flip with transform-style: preserve-3d — Athlete cards flip 180° on focus-within, using perspective, backface-visibility: hidden, and rotateY(180deg).
  4. Table-to-card reflow via nth-child + ::before — Each <td> gets a labeled pseudo-element based on column position, without needing data- attributes in HTML.
  5. Horizontal scroll-snap gallery — scroll-snap-type: x mandatory with grid-auto-flow: column creates a swipeable image carousel.
  6. Frosted glass effect — Athlete records badge uses backdrop-filter: blur(8px) over the gradient header.
  7. Custom select dropdown arrow — Inline SVG data URI replaces the browser default: background-image: url("data:image/svg+xml,...").
  8. Animated progress bars — Accomplishment items feature decorative gradient bars that animate from scaleX(0) to scaleX(1) with staggered delays.
  9. Floating Action Button (FAB) — Fixed-position back-to-top button with smooth scroll, hover lift effect, and circular styling.
  10. Gradient header with animated entrance — Linear gradient background with fadeInDown animation on the heading.
  11. Section heading accent underlines — Gold decorative bar beneath each <h2> via ::after pseudo-element.
  12. Alternating table-card border colors — :nth-child(even/odd) applies different left-border accent colors to table row cards.
  13. Custom cubic-bezier easing — Named easing variables (--ease-out, --ease-in-out) for polished animation feel.
  14. Staggered animation delays — Accomplishment items and comparison cards enter sequentially using incremented animation-delay.
  15. Subtle pulse glow on CTA button — Compare button has an infinite subtlePulse box-shadow animation to draw attention.
  16. Touch-active press feedback — Gallery images and accomplishment cards scale down slightly on :active for tactile response.

3. Animations

10 pts

Multiple CSS animations enhance the user experience without being distracting. All animations respect prefers-reduced-motion.

Animations Implemented

  1. fadeIn — Smooth opacity entrance for profile photos and performance graph images.
  2. fadeInUp — Content sections, table row cards, and accomplishment items slide up as they appear, creating a sense of upward momentum.
  3. fadeInDown — Page header title drops in from above for a dynamic first impression.
  4. slideInLeft — Comparison cards slide in from the left with staggered timing (150ms delay between cards).
  5. progressFill — Decorative gradient bars beneath each accomplishment animate from scaleX(0) to scaleX(1) with transform-origin: left, creating a filling progress bar effect.
  6. subtlePulse — The Compare button features an infinite 3-second pulse glow (box-shadow expanding and fading) to signal interactivity.
  7. 3D Card Flip — Athlete roster cards rotate 180° around the Y-axis when focused, revealing stats on the back. Uses transform-style: preserve-3d, backface-visibility: hidden, and a 600ms ease-in-out transition.
  8. Staggered entrance delays — Accomplishment items enter sequentially (100ms–600ms delays), creating a cascading reveal effect.
  9. Touch press feedback — Gallery images and list items scale to 97–98% on :active, providing immediate tactile feedback on touch.
  10. Button hover/focus lift — Buttons and links translate up 1–2px on hover/focus with shadow enhancement, creating a subtle lift illusion.

Code Sample — Keyframes

  from { opacity: 0; transform: translateY(1.5rem); }
  to   { opacity: 1; transform: translateY(0); }

@keyframes progressFill {

  to   { transform: scaleX(1); }

@keyframes subtlePulse {

}

6. Addresses Prefers-Reduced-Motion

5 pts

A @media (prefers-reduced-motion: reduce) query disables all animations and transitions for users who have enabled "Reduce motion" in their OS accessibility settings.

Implementation

/* Section 20 of main.css */
@media (prefers-reduced-motion: reduce) {
  *,
  *::before,
  *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
    scroll-behavior: auto !important;
  }
}

This universal rule catches every animation and transition on the page — including the card flip, progress bars, fade-ins, and the pulse glow — and effectively disables them all by setting durations to near-zero. scroll-behavior is also reset to auto so smooth scrolling does not trigger vestibular issues.

This complies with WCAG 2.1 Level AA, Guideline 2.3.3 (Animation from Interactions).

7. Addresses User Color Scheme Preference

5 pts

A @media (prefers-color-scheme: dark) query provides a full dark-mode experience that activates automatically based on the user's OS or browser setting.

Implementation

All colors are defined as CSS custom properties in :root. The dark mode media query overrides only the variable values, so every element that references them automatically adapts:

/* Light mode (default) */
:root {
  --clr-primary: #1b3a5c;
  --clr-bg: #f4f1ec;
  --clr-surface: #ffffff;
  --clr-text: #1a1a2e;
  /* ... */
}

/* Dark mode */
@media (prefers-color-scheme: dark) {
  :root {
    --clr-primary: #4a8ac4;
    --clr-bg: #121220;
    --clr-surface: #1e1e32;
    --clr-text: #e4e4ec;
    /* ... */
  }
}

What Changes in Dark Mode

  • Page background shifts from warm light grey (#f4f1ec) to deep navy (#121220)
  • Card surfaces become dark (#1e1e32) with adjusted shadows for depth
  • Text becomes light (#e4e4ec) with muted secondary text (#9a9ab0)
  • Primary accent shifts to a brighter blue (#4a8ac4) for visibility
  • Links use brighter blue (#6aaae4) and gold (#e0b44a) for hover
  • Shadows increase in opacity to remain visible on dark backgrounds
