Project Overview

This project is a mobile-only website for the Ann Arbor Skyline Cross Country team. It consists of a Team Page with the athlete roster, accomplishments, comparison tools, event results, and a photo gallery, as well as an individual Athlete Page for Garrett Comer with his profile, race results, performance graph, and race photos. All styling is in a single custom stylesheet organized into 20 commented sections.


Mobile-First Design

The site is built entirely as a mobile-only layout with no width-based media queries and no use of the max-width property anywhere in the CSS. All content stacks in a single vertical column using Flexbox, optimized for thumb-scrolling on mobile devices. The navigation bar is sticky at the top and supports horizontal swiping for the tab links. Font sizes scale fluidly with the viewport using the clamp() function, so text remains readable at any screen width without breakpoints. Tables are reflowed into stacked card layouts where each row becomes its own card with labeled fields, and the gallery uses scroll-snap for a native swipeable carousel feel.


CSS Quality

The stylesheet is organized into 20 clearly labeled sections with a table of contents at the top. Over 25 CSS custom properties are defined in the root element for colors, backgrounds, spacing, shadows, radii, and timing, which eliminates repetitive code and enables the dark mode to work by simply overriding those variables. Background-specific variables (--clr-nav-bg, --clr-header-start, --clr-header-end) are decoupled from the primary accent color so that dark mode can use a bright blue for text and a very dark navy for backgrounds simultaneously, ensuring WCAG AA contrast compliance in both color schemes.

Innovative techniques include 3D card flips using preserve-3d and backface-visibility, table-to-card reflow using nth-child selectors with ::before pseudo-element labels, a frosted glass effect with backdrop-filter: blur on the athlete records badge, a custom select dropdown arrow via an inline SVG data URI, animated progress bars with staggered delays on the accomplishments, a floating action button for back-to-top navigation, decorative heading underlines via ::after pseudo-elements, alternating left-border accent colors on table cards, custom cubic-bezier easing curves, and a subtle infinite pulse glow on the Compare button.


Animations

The site includes multiple CSS animations that enhance the experience without being distracting. Content sections and cards use fadeInUp to slide into view, the page title uses fadeInDown, and comparison cards use slideInLeft with staggered timing. Accomplishment items have animated progress bars that fill from left to right using scaleX with staggered delays. The Compare button has a subtle infinite pulse glow on its box-shadow. The most prominent animation is the 3D card flip on the roster cards, which rotates 180 degrees on tap or keyboard focus to reveal athlete stats on the back. Gallery images and list items also have touch press feedback, scaling down slightly on active to simulate a physical press. All animations are fully disabled when the user prefers reduced motion.

Addresses Prefers-Reduced-Motion Preference

The stylesheet includes a @media (prefers-reduced-motion: reduce) query that targets all elements using the universal selector. It sets animation-duration and transition-duration to near-zero, limits animation-iteration-count to one, and resets scroll-behavior to auto. This single rule disables every animation and transition on the site, including the card flip, progress bars, fade-ins, and the button pulse, ensuring users with motion sensitivities are not affected.


Addresses User's Color Scheme Preference

The site supports automatic dark mode through a @media (prefers-color-scheme: dark) query. All colors are defined as CSS custom properties, so the dark mode query only needs to redefine the variable values in one place. The color system uses separate variables for backgrounds that need to stay dark for white text contrast (--clr-nav-bg, --clr-header-start, --clr-header-end) versus the primary accent color which needs to stay bright for text on dark surfaces. When dark mode activates, the page background shifts to deep navy, card surfaces become dark slate, text becomes off-white, the navigation and footer backgrounds become an even darker navy for strong white text contrast, and shadow opacity increases. Interactive elements like the Compare button and View Profile link use a hardcoded dark text color to maintain readability on their gold backgrounds regardless of color scheme. Every element on the site adapts automatically without any duplicate rules.


No Console Errors

The site produces zero console errors. There are no script tags on either page, both CSS files load with correct relative paths, there are no inline style attributes in the HTML, and all external links that open in new tabs include rel="noopener".


Presentation and Clarity

The Team Page includes five sections: an athlete roster with flip cards, accomplishments with progress bars, an athlete comparison tool with stat cards and a shared meets table, team events in a card-reflow layout, and a horizontal scroll-snap photo gallery. The Athlete Page features a profile header with a circular photo and frosted-glass records badge, race results in labeled cards, a performance graph, and a photo gallery.

Accessibility features include a skip-to-content link on every page, a proper heading hierarchy with no skipped levels, alt text on all images, labels on all form controls, aria-describedby on inputs, aria-label on the back-to-top button, a visible focus ring for keyboard navigation, visually hidden headings preserved for screen readers, semantic HTML landmarks throughout, and lang="en" on all pages.
